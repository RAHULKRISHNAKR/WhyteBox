# WhyteBox Frontend - 3D Neural Network Visualization System

## Overview

A modular BabylonJS-based 3D visualization system that renders neural network models from backend JSON metadata. The system uses a registry pattern where each layer type has a dedicated visualization function, allowing for extensible and customizable 3D representations.

## Features

✅ **Multi-Layer Support**: Visualizes Conv2d, Dense, Pooling, Activation, Normalization, and more  
✅ **Dynamic Layouts**: Automatic layout selection (Linear for VGG, Hierarchical for ResNet)  
✅ **Connection Rendering**: Sequential (straight), Skip (curved), and Branch (multi-target) connections  
✅ **Interactive 3D**: Orbit controls, layer selection, hover effects  
✅ **Layer Expansion**: Click any Conv/Pool/Dense layer to expand and view feature maps  
✅ **Hover Details**: Mouse over layers to see detailed information (shapes, parameters, etc.)  
✅ **Backend Integration**: Direct loading from WhyteBox backend JSON format  
✅ **Modular Architecture**: Easy to extend with new layer types and visualizers

## Quick Start

### 1. Prerequisites

- Modern web browser with WebGL support
- Backend JSON model files (generated by WhyteBox backend)
- Local web server (for loading files, e.g., `python -m http.server` or Live Server extension)

### 2. Open the Example

```bash
cd frontend
# Start a local server (Python 3)
python -m http.server 8000

# Or use Python 2
python -m SimpleHTTPServer 8000

# Open in browser
# http://localhost:8000/examples/visualize-model.html
```

### 3. Load a Model

**Option A: Demo Models**
- Click "Load VGG16" button
- Requires backend to have generated `backend/output/vgg16_pytorch_visualization.json`

**Option B: Custom Model**
- Click "Choose JSON file..." button
- Select a visualization JSON file from your backend output

## Architecture

### Core Components

```
frontend/
├── src/
│   ├── core/
│   │   └── Scene3DManager.js           # BabylonJS scene, camera, lighting
│   ├── loaders/
│   │   └── ModelLoader.js               # Load & validate backend JSON
│   ├── visualization/
│   │   ├── LayerVisualizerRegistry.js   # Registry for layer visualizers
│   │   ├── ModelVisualizer.js           # Main orchestrator
│   │   ├── ConnectionRenderer.js        # Render connections
│   │   ├── layerVisualizers/            # Individual layer visualizers
│   │   │   ├── ConvolutionalVisualizer.js
│   │   │   ├── PoolingVisualizer.js
│   │   │   ├── DenseVisualizer.js
│   │   │   ├── ActivationVisualizer.js
│   │   │   ├── NormalizationVisualizer.js
│   │   │   ├── UtilityVisualizer.js
│   │   │   ├── InputOutputVisualizer.js
│   │   │   └── DefaultVisualizer.js
│   │   └── layouts/
│   │       ├── LinearLayout.js          # Sequential arrangement
│   │       └── HierarchicalLayout.js    # Level-based arrangement
│   └── utils/
│       ├── ColorUtils.js                # Color parsing & defaults
│       └── GeometryUtils.js             # Geometry creation utilities
└── examples/
    └── visualize-model.html             # Example application
```

### Workflow

1. **Load Model**: `ModelLoader` fetches and validates JSON
2. **Initialize Scene**: `Scene3DManager` sets up BabylonJS scene
3. **Setup Registry**: Register all layer visualizers
4. **Calculate Layout**: Choose layout algorithm (Linear/Hierarchical)
5. **Create Meshes**: For each layer, lookup visualizer and create 3D mesh
6. **Render Connections**: Draw lines/curves between layers
7. **Apply Camera**: Position camera based on hints
8. **Render Loop**: Start BabylonJS render loop

## Backend JSON Format

The system expects JSON in the following format:

```json
{
  "metadata": {
    "model_name": "VGG16",
    "framework": "pytorch",
    "total_layers": 41,
    "total_parameters": 138357544
  },
  "layers": [
    {
      "id": "layer_0",
      "name": "features.0",
      "type": "Conv2d",
      "index": 0,
      "input_shape": [1, 3, 224, 224],
      "output_shape": [1, 64, 224, 224],
      "parameters": { "in_channels": 3, "out_channels": 64, ... },
      "visualization": {
        "color": "#4A90E2",
        "size_hint": 1.5
      }
    }
  ],
  "connections": [
    {
      "source_layer": "layer_0",
      "target_layer": "layer_1",
      "connection_type": "sequential",
      "visualization": {
        "color": "#999999",
        "width": 2.0
      }
    }
  ],
  "visualization_hints": {
    "suggested_layout": "linear",
    "camera_position": [0, 5, 20],
    "layer_spacing": 2.0
  },
  "topology": {
    "input_layers": ["layer_0"],
    "output_layers": ["layer_40"]
  }
}
```

## Layer Visualization Shapes

| Layer Type | 3D Shape | Sizing Logic |
|------------|----------|--------------|
| **Conv2d** | Rectangular prism | Height ∝ output channels |
| **Linear/Dense** | Tall column | Height ∝ output units |
| **MaxPool2d** | Flatter prism | 50% of Conv height |
| **ReLU** | Small icosahedron | Fixed small size, colored red |
| **BatchNorm** | Thin transparent box | 10% height, semi-transparent |
| **Flatten** | Wireframe cylinder | Minimal transition marker |
| **Dropout** | Very transparent box | Same size as input |
| **Input** | Green bordered cube | Special glow effect |
| **Output** | Orange bordered box | Height ∝ output classes |

### Color Scheme

- **Conv2d**: Blue (`#4A90E2`)
- **Dense**: Green (`#50C878`)
- **Pooling**: Orange (`#F5A623`)
- **BatchNorm**: Purple (`#BD10E0`)
- **Activation**: Red (`#FF6B6B`)
- **Utility**: Gray (`#95A5A6`)

## Usage Example

### Basic Usage

```javascript
// Create visualizer
const visualizer = new ModelVisualizer();

// Load and visualize from URL
await visualizer.visualizeModel(
    '../backend/output/vgg16_pytorch_visualization.json',
    'renderCanvas'
);

// Get metadata
const metadata = visualizer.getMetadata();
console.log(`Model: ${metadata.model_name}`);

// Toggle connections
visualizer.toggleConnections(false); // Hide connections

// Focus on a layer
visualizer.focusOnLayer('layer_5');

// Reset camera
visualizer.resetCamera();

// Dispose when done
visualizer.dispose();
```

### Programmatic Model Creation

```javascript
const visualizer = new ModelVisualizer();

// Create model data programmatically
const modelData = {
    metadata: {
        model_name: "CustomModel",
        framework: "pytorch",
        total_layers: 3,
        total_parameters: 1000
    },
    layers: [
        {
            id: "layer_0",
            name: "input",
            type: "Input",
            index: 0,
            output_shape: [1, 3, 224, 224]
        },
        {
            id: "layer_1",
            name: "conv1",
            type: "Conv2d",
            index: 1,
            output_shape: [1, 64, 224, 224],
            visualization: { color: "#4A90E2" }
        },
        {
            id: "layer_2",
            name: "output",
            type: "Output",
            index: 2,
            output_shape: [1, 10]
        }
    ],
    connections: [
        { source_layer: "layer_0", target_layer: "layer_1", connection_type: "sequential" },
        { source_layer: "layer_1", target_layer: "layer_2", connection_type: "sequential" }
    ],
    visualization_hints: {
        suggested_layout: "linear",
        layer_spacing: 2.5
    }
};

// Visualize
await visualizer.visualizeModel(modelData, 'renderCanvas');
```

## Extending the System

### Adding a New Layer Type

1. **Create Visualizer Function**

```javascript
// src/visualization/layerVisualizers/CustomVisualizer.js
const CustomVisualizer = {
    visualizeMyCustomLayer(layer, scene, position, options = {}) {
        const dimensions = { width: 1, height: 2, depth: 1 };
        const color = ColorUtils.getDefaultLayerColor('MyCustomLayer');
        
        const mesh = GeometryUtils.createLabeledBox(
            layer.id,
            dimensions,
            color,
            scene
        );
        
        mesh.position = position.clone();
        
        const connectionPoints = GeometryUtils.getConnectionPoints(mesh, dimensions);
        
        return {
            mesh: mesh,
            bounds: dimensions,
            connectionPoints: connectionPoints
        };
    }
};
```

2. **Register in ModelVisualizer**

In `ModelVisualizer._setupRegistry()`:

```javascript
this.registry.register('MyCustomLayer', 
    CustomVisualizer.visualizeMyCustomLayer.bind(CustomVisualizer)
);
```

3. **Add Default Color** (optional)

In `ColorUtils.getDefaultLayerColor()`:

```javascript
'MyCustomLayer': '#FF00FF', // Magenta
```

### Creating a Custom Layout

```javascript
// src/visualization/layouts/RadialLayout.js
class RadialLayout {
    static calculatePositions(layers, hints = {}) {
        const positions = new Map();
        const radius = 10;
        const angleStep = (2 * Math.PI) / layers.length;
        
        layers.forEach((layer, index) => {
            const angle = index * angleStep;
            const position = new BABYLON.Vector3(
                radius * Math.cos(angle),
                0,
                radius * Math.sin(angle)
            );
            positions.set(layer.id, position);
        });
        
        return positions;
    }
}
```

4. **Hardware Acceleration**: Ensure GPU acceleration is enabled in browser settings

## Troubleshooting

### Model Won't Load

- ✅ Check JSON format matches backend specification
- ✅ Verify file path is correct
- ✅ Use a local server (required for CORS)
- ✅ Check browser console for errors

### Black Screen

- ✅ Ensure BabylonJS CDN is loading
- ✅ Check WebGL is supported (`chrome://gpu/`)
- ✅ Verify canvas element exists

### Layers Not Rendering

- ✅ Check layer types are registered
- ✅ Verify layer visualizers are included in HTML
- ✅ Check console for missing visualizer warnings
